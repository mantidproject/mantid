- :ref:`Stitch1D <algm-Stitch1D>` and :ref:`Stitch1DMany <algm-Stitch1DMany>` now have a ``UseValidDataOnly`` option which, when ``true``, ignores the contribution of invalid signal values to overlap bins when another workspace has valid data in the same bin. When ``false`` (default) previous behaviour is maintained, where presence of invalid data in an overlap bin from either workspace results in the same invalid value in the output workspace.
