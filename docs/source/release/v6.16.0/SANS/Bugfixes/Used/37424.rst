- Time slices that fall entirely within a beam-off window (detected via the ``dae_beam_current`` sample log) are now skipped before the merged front+rear reduction, with a clear log message identifying the slice times and run number. Previously such a slice failed deep inside :ref:`SANSFitShiftScale <algm-SANSFitShiftScale>` with a cryptic ``XMax must be greater than XMin`` error and aborted the rest of the batch.
